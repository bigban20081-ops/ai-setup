# Web Page Architect

## Overview

Объединяет три роли при создании веб-страницы: **маркетолог** (конверсия, SEO, копирайтинг), **дизайнер** (UI/UX, иерархия, адаптивность), **фронтенд-разработчик** (семантика, производительность, доступность).

**Принцип:** Каждая страница — это воронка. Каждый элемент должен либо продвигать пользователя к цели, либо снимать возражение.

## When to Use

- Создание новой страницы (landing, product, pricing, about, docs, blog)
- Редизайн существующей страницы
- Аудит конверсии/SEO/UX существующей страницы
- Написание копирайтинга для веб-страницы

**НЕ использовать для:** email-рассылок, мобильных приложений (native), печатных материалов.

## Core Process

### Phase 0: Контекст (ОБЯЗАТЕЛЬНО перед дизайном)

Прежде чем рисовать — ответь на 5 вопросов:

| Вопрос | Зачем |
|--------|-------|
| **Кто** целевая аудитория? | Определяет тон, боли, возражения |
| **Что** они должны сделать на странице? | Определяет CTA и воронку |
| **Почему** они должны выбрать нас? | Определяет УТП и позиционирование |
| **Откуда** они придут? | Определяет контекст (реклама, SEO, прямой) |
| **Какие возражения** у них будут? | Определяет структуру social proof и FAQ |

Если заказчик не дал ответы — задай вопросы. Не выдумывай ЦА.

### Phase 1: Конверсионная архитектура

Выбери модель воронки по типу страницы:

**Landing (SaaS/Product):**
```
Hero (УТП + CTA) → Social Proof (логотипы) → Проблема (боль) →
Решение (фичи) → Как работает (шаги) → Доказательства (метрики + отзывы) →
Сравнение (таблица) → Тарифы → FAQ → Финальный CTA
```

**Product Feature page:**
```
Hero (фича + визуал) → Проблема → Решение (демо/скриншоты) →
Технические детали → Связанные фичи → CTA
```

**About/Company:**
```
Миссия → Команда → Цифры → Ценности → Партнеры → CTA (карьера/контакт)
```

**Blog/Docs:**
```
Заголовок + мета → Содержание (TOC) → Контент с H2/H3 →
Related articles → CTA (подписка/продукт)
```

### Phase 2: Маркетинг и копирайтинг

#### Заголовки (критично для конверсии)

**H1 — один на страницу:**
- Конкретный результат, не абстракция
- Содержит цифру или метрику если возможно
- Max 10 слов

| Плохо | Хорошо |
|-------|--------|
| "Управляйте задачами эффективно" | "AI ревьюит PR за 30 секунд" |
| "Лучший инструмент для команд" | "Команды закрывают проекты на 40% быстрее" |
| "Ваш надежный партнер" | "4.6 млрд выручки выпускников за 2 года" |

**H2 — один на секцию:**
- Отвечает на вопрос "почему мне это важно?"
- Может быть вопросом ("Знакомо?") или утверждением ("Код остается у вас")

#### CTA-кнопки

- **Primary CTA** повторяется min 3 раза: header, hero, финальный блок
- **Текст CTA** = действие + результат, НЕ "Узнать больше"
- **Под CTA** — снятие возражения мелким текстом

| Плохо | Хорошо |
|-------|--------|
| "Отправить" | "Начать бесплатно" |
| "Узнать больше" | "Попробовать 14 дней бесплатно" |
| "Подписаться" | "Получить отчет на email" |

Под кнопкой: "Без карты. Настройка за 2 минуты."

#### Social Proof (иерархия доверия)

Размещай от сильного к слабому, чем раньше — тем лучше:

1. **Метрики** (цифры клиентов, выручка, uptime) — в hero или сразу после
2. **Логотипы** (клиенты/партнеры) — сразу после hero
3. **Отзывы** (с фото, должностью, компанией) — после фич
4. **Сравнительная таблица** — перед pricing
5. **Сертификаты/бейджи** — footer или рядом с pricing

Правила:
- Цифры всегда конкретные: "3 247 команд", не "тысячи клиентов"
- Отзывы с фото, именем, должностью и компанией — иначе не верят
- Если нет реальных метрик — не выдумывай, лучше пропусти секцию

#### Urgency и Scarcity

Используй только когда есть реальный повод:
- Дедлайн приема заявок (с датой)
- Ограниченный trial ("7 дней Pro бесплатно")
- Скидка при годовой оплате ("Экономия 20%")

НЕ используй fake urgency ("Осталось 3 места!" без реального лимита).

### Phase 3: Дизайн и визуальная иерархия

#### Цветовая схема

**Правило 60-30-10:**
- 60% — фон (neutral)
- 30% — secondary (поверхности, карточки)
- 10% — accent (CTA, акценты, ссылки)

**Тёмная vs светлая тема:**

| Темная | Светлая |
|--------|---------|
| Tech, dev-tools, AI, premium | Корп, здоровье, образование, ecom |
| Фон: #070626-#0F172A | Фон: #FFFFFF, alt: #F8FAFC |
| Акценты ярче (неон, градиент) | Акценты приглушённее |

**Обязательно:** контраст текст/фон min 4.5:1 (WCAG AA). Проверяй.

#### Типографика

- **Один font-family** для всей страницы (max 2). Не Inter для каждого проекта — выбирай под характер бренда
- **H1:** 48-80px desktop / 32-40px mobile, weight 700-800
- **H2:** 32-48px / 24-32px, weight 600-700
- **Body:** 16-18px / 16px, line-height 1.5-1.7
- **Small:** 14px, для подписей и disclaimers

#### Spacing система

```
Между секциями: 80-120px desktop / 48-64px mobile
Заголовок → контент: 32-48px / 24-32px
Между карточками: 24-32px / 16-24px
Padding карточек: 24-40px
Container max-width: 1200-1280px, padding-x: 16-24px
```

#### Above the Fold (критично)

Первый экран (до скролла) ДОЛЖЕН содержать:
1. H1 с УТП
2. Подзаголовок (1-2 предложения)
3. Primary CTA
4. Один визуальный элемент (скриншот, иллюстрация, видео)
5. Социальное доказательство (логотипы или ключевая метрика)

Ничего лишнего. Пользователь решает за 3-5 секунд — остаться или уйти.

#### Адаптивность (min 375px)

```
Desktop: >= 1024px (сетки 2-4 колонки)
Tablet:  768-1023px (сетки 2 колонки)
Mobile:  375-767px (1 колонка, кнопки width:100%)
```

Правила:
- Mobile-first: сначала дизайн для 375px, потом расширяй
- Кнопки на мобильных — на всю ширину
- Горизонтальные сетки складываются вертикально
- Текст не меньше 16px на мобильных (iOS zoom prevention)
- Touch targets min 44x44px

#### Анимации

- **Цель:** направить внимание, не развлекать
- Fade-in при скролле — для секций (stagger 100-150ms между элементами)
- Hover-эффекты — для карточек и кнопок (scale 1.02-1.05, 200ms ease)
- **НЕ анимируй** текст параграфов, навигацию, footer
- **prefers-reduced-motion:** всегда уважай

### Phase 4: SEO и техническая оптимизация

#### Мета-теги (обязательный минимум)

```html
<title>Ключевое слово — Бренд | Дополнение (max 60 символов)</title>
<meta name="description" content="... (max 155 символов, с CTA)">
<meta name="robots" content="index, follow">
<link rel="canonical" href="https://...">

<!-- Open Graph -->
<meta property="og:title" content="...">
<meta property="og:description" content="...">
<meta property="og:image" content="...(1200x630)">
<meta property="og:type" content="website">
<meta property="og:url" content="...">

<!-- Twitter -->
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="...">
<meta name="twitter:description" content="...">
<meta name="twitter:image" content="...">
```

#### Schema.org (по типу страницы)

| Тип страницы | Schema |
|-------------|--------|
| SaaS/Product | SoftwareApplication + Organization + FAQPage |
| Услуги | Service + Organization + FAQPage |
| Блог | Article + BreadcrumbList |
| About | Organization + Person (команда) |
| Pricing | Offer + AggregateRating |

#### Семантическая разметка

```html
<header>     <!-- навигация, 1 на страницу -->
<main>       <!-- основной контент, 1 на страницу -->
  <section>  <!-- каждая секция страницы -->
<footer>     <!-- подвал, 1 на страницу -->
<nav>        <!-- навигация -->
<article>    <!-- самостоятельный контент (блог-посты) -->
<aside>      <!-- боковой контент -->
```

- Один `<h1>` на страницу
- `<h2>` для секций, `<h3>` для подразделов — не пропускай уровни
- Изображения: всегда `alt` (описательный, не "image.png")
- Ссылки: осмысленный текст, не "нажмите здесь"

#### Performance (Web Vitals бюджет)

| Метрика | Target | Как |
|---------|--------|-----|
| LCP | < 2.5s | Оптимизация hero image, preload критических ресурсов |
| CLS | < 0.1 | Размеры для img/video, font-display: swap |
| INP | < 200ms | Минимум JS на первом экране |

Обязательно:
- Изображения: WebP/AVIF, lazy loading (кроме hero), srcset для responsive
- Шрифты: font-display: swap, preload для основного шрифта
- CSS: критический инлайном, остальной — async
- JS: defer/async, минимум в `<head>`

#### Accessibility (WCAG 2.1 AA)

- Контраст текст/фон: min 4.5:1 (обычный текст), 3:1 (крупный)
- Focus states для всех интерактивных элементов
- `aria-label` для иконок-кнопок без текста
- Skip-to-content ссылка
- Формы: `<label>` для каждого input, error messages связаны через aria-describedby
- Не полагайся только на цвет для передачи информации

### Phase 5: Аналитика

Для каждой страницы определи events:

```
// Обязательные
page_view           — загрузка страницы
cta_click           — клик по любому CTA (с label какого именно)
scroll_depth        — 25%, 50%, 75%, 100%

// По типу страницы
pricing_view        — просмотр тарифов
faq_expand          — открытие вопроса FAQ
demo_request        — запрос демо
signup_start        — начало регистрации
video_play          — воспроизведение видео
```

## Quick Reference: Чеклист перед публикацией

### Маркетинг
- [ ] ЦА определена, боли и возражения описаны
- [ ] H1 конкретный, с результатом/метрикой
- [ ] CTA повторяется min 3 раза, текст = действие + результат
- [ ] Social proof в первых 2 экранах
- [ ] Каждая секция снимает возражение или продвигает к CTA
- [ ] Urgency только при реальном поводе

### Дизайн
- [ ] Цветовая схема 60-30-10
- [ ] Контраст min 4.5:1
- [ ] Above the fold: H1 + CTA + визуал + social proof
- [ ] Адаптивность от 375px
- [ ] Spacing консистентный (система)
- [ ] Анимации направляют внимание, не отвлекают

### SEO
- [ ] title (60 символов), description (155 символов)
- [ ] OG и Twitter мета-теги
- [ ] Schema.org по типу страницы
- [ ] Один h1, иерархия h2-h3 без пропусков
- [ ] Canonical URL
- [ ] alt для всех изображений

### Техника
- [ ] Семантический HTML (header, main, section, footer)
- [ ] Web Vitals: LCP < 2.5s, CLS < 0.1, INP < 200ms
- [ ] Изображения: WebP, lazy load, srcset
- [ ] Шрифты: font-display: swap, preload
- [ ] Accessibility: focus states, aria-labels, skip-to-content
- [ ] Analytics events настроены

### Копирайтинг
- [ ] Заголовки: результат > процесс, конкретика > абстракция
- [ ] Тексты: короткие абзацы (max 3 строки), простые слова
- [ ] Benefits > Features (не "AI-алгоритм", а "находит баги за 30 секунд")
- [ ] Цифры конкретные ("3 247 команд", не "тысячи")

## Common Mistakes

| Ошибка | Почему плохо | Как правильно |
|--------|-------------|---------------|
| Дизайн без анализа ЦА | Страница не резонирует, низкая конверсия | Phase 0 обязательна |
| Generic H1 ("Лучшее решение") | Не цепляет, не отличает от конкурентов | Конкретный результат + метрика |
| CTA "Узнать больше" | Не мотивирует к действию | "Попробовать бесплатно" / "Начать за 2 минуты" |
| Social proof после 3-го экрана | Пользователь уже ушёл | Логотипы/метрики сразу после hero |
| Inter + синий + белый фон | Выглядит как шаблон, нет характера | Выбирай палитру и шрифт под бренд |
| Fake urgency | Подрывает доверие | Только реальные дедлайны и лимиты |
| H1, H3, H2 (нарушение иерархии) | Плохо для SEO и a11y | Строгая иерархия h1 > h2 > h3 |
| Нет performance-бюджета | Медленная страница = потеря конверсий | Web Vitals targets с Phase 4 |
| Нет мобильной версии | 60%+ трафика с мобильных | Mobile-first design от 375px |
| Отзывы без лиц и должностей | Не вызывают доверия | Фото + имя + должность + компания |
